1. 우리의 서비스를 사용하고자 하는 구글에다가 등록한다
2. 구글로 부터 서비스의 id 와 secret key와 같은 것을 제공받아 
   구글와 우리의 서비스 모두 가지고 있다.
3. 우리가 사용자에게 구글로 로그인 하는 button이라던지 page를 보여주면
   그 url 뒤에 우리의 service id, scope	와 같은 것들이 첨부되어서 넘어가게된다

4. 이때 로그인에 성공하게 되면 
   구글은 클라이언트 id와 secret에 대하여
   해당 user_id 가 로그인을 했고 인증을 해준다는 차원에서
    authorizetion_code를 부여해준다
5. 그럼 사용자는 로그인이 되는순가
   authorizetion code를 받아
   client에게 전달하게 되고
   client는 제공받은 code를 resource server와 비교해
   여러 정보들이 일치하게 되면 accessToken을 구글로 부터 제공받는다
   그럼 이후에 구글에 접근하여 사용자의 기능 및 정보를 받아올때는 access token을 사용하여비교한다


사용자의 친구를 보여달라는 기능을 요청하면
우리는 트위터에 요청을 하고 트위터는 시크릿 키와 라던지 request요청에 대해 다시 반환하는데
우리는 그것을 사용자에게 url로 클릭하여 인증을 받을수있게 전달하고
사용자는 그것을 눌러 동의를 하고나면
access token을 다시금 사용자에게 제공해주고 그걸 제공받은 사용자는 redirect를 통해 우리의 서비스 내에서
우리가 트위터에 사용자가 친구요청을 할수 있게 요청승인을 해준다.
